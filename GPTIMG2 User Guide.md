## How to Write Prompts That Work

Forget “prompt hacks.” Think like you're giving instructions to a designer.

---

### Good Prompt Structure

Use this format:

**1. Goal**  
What are you trying to create?

**2. Constraints**  
Position, colors, style, layout

**3. Details**  
Lighting, mood, textures

---

#### Example (Effective)

> Create a modern product banner for a smartwatch. Place the watch centered, add a soft shadow underneath, use a dark gradient background, and include the text “Stay Connected” at the top in clean sans-serif font.

#### Example (Weak)

> cool smartwatch ad

### Key Principle

> The more specific your intent, the better GPTIMG2 performs.

## Common Problems (And How to Fix Them)

No tool is perfect. Here’s what users consistently run into.

---

### Problem 1: Text still looks slightly off

**Fix:**
- Keep text short  
- Avoid complex fonts  
- Regenerate only the text area if possible  

---

### Problem 2: Layout drift

Sometimes elements shift unexpectedly.

**Fix:**
- Specify exact positions  
- Use phrases like:
  - “top-left corner”
  - “center aligned”
  - “equal spacing”

---

### Problem 3: Over-interpretation

The model adds things you didn’t ask for.

**Fix:**
- Add constraints:
  - “do not add extra elements”
  - “keep background minimal”

---

### Problem 4: Inconsistent style across iterations

**Fix:**
- Reuse the same base image  
- Avoid rewriting prompts completely
